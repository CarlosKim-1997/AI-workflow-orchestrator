# 발표 자료 제작용 프롬프트

> 아래 내용을 **발표 슬라이드/PT 자료를 만들어 주는 AI**에게 그대로 붙여 넣어 사용하세요.  
> 프로젝트 스크린샷을 넣을 슬라이드는 가운데에 이미지 영역만 비워 두라고 명시되어 있습니다.

---

## 프롬프트 (복사용)

```
다음은 포트폴리오용으로 소개할 **AI 워크플로우 오케스트레이터** 프로젝트입니다.  
이 프로젝트는 **테스트/데모용 PoC(개념 증명)** 로 만들었고, 실제 동작을 검증하려면 OpenAI, Jira, Slack, 이메일 등 여러 외부 서비스의 계정과 API 키가 필요합니다.  
그래서 발표에서는 “무엇을 염두에 두고 만들었는지”, “어떤 기능과 구조를 갖췄는지”를 **포트폴리오 형식**으로 설명하는 자료를 만들어 달라고 요청합니다.

---

### 1. 프로젝트 개요 (1~2장)

- **제목**: AI 워크플로우 오케스트레이터 — 엔터프라이즈 AI 트랜스포메이션 PoC
- **한 줄 요약**: 고객 이메일을 AI가 분석해 의도·부서·우선순위를 판단하고, Jira·Slack·이메일 등과 연동해 워크플로우를 자동 생성·실행하는 데모 애플리케이션
- **목적**: 대기업 고객센터/지원팀처럼 이메일 유입이 많은 환경에서 “분류·라우팅·시스템 입력”을 자동화하는 방안을 **시연·검증**하기 위함
- **성격**: 실제 운영용이 아니라 **테스트·PoC·포트폴리오**용이며, 실제 연동 테스트에는 다수의 API 키와 계정이 필요하다는 점을 슬라이드에 한 줄 정도로 명시해 주세요.

---

### 2. 프로젝트 화면 (1장) — 여기에 스크린샷 배치

**이 슬라이드 한 장은 반드시 다음 형태로 만들어 주세요.**

- 슬라이드 제목: 「프로젝트 화면」 또는 「시스템 구성 예시」
- 슬라이드 **가운데**에 **프로젝트 스크린샷을 넣을 빈 영역**을 두세요.  
  즉, 이미지를 넣지 말고, “여기에 프로젝트 스크린샷을 넣을 영역”이라고 표시된 **빈 박스/플레이스홀더**만 넣어 주세요.
- 필요하면 박스 안에 작게 「스크린샷 배치 위치」 같은 안내 문구를 넣어도 됩니다.
- 이 슬라이드 아래나 옆에 짧은 캡션만 넣어도 됩니다. 예: “Streamlit 기반 웹 UI — 이메일 입력, 분석 결과, 워크플로우 실행 화면”

---

### 3. 해결하려는 문제 (1장)

- 고객센터/지원팀에서 **하루 수백 건** 이메일 유입 시, 담당자가 수동으로 분류하는 데 시간이 많이 듦
- 부서 간 이관 시 **담당자 파악 지연**
- Jira, Slack, 메일 등 **여러 시스템에 각각 수동 입력**해야 함
- 긴급 건이 **주관적 판단**으로 누락될 위험

→ “AI가 이메일을 분석해 의도·부서·우선순위를 정하고, 워크플로우를 자동 생성·실행한다”는 솔루션 방향을 한 장에 요약해 주세요.

---

### 4. 핵심 기능 (1~2장)

- **AI 이메일 분석**: GPT 기반으로 이메일에서 의도(Intent), 담당 부서, 우선순위, 신뢰도 추출
- **동적 워크플로우 생성**: 의도·키워드에 따라 “카드 분실”, “긴급 발주”, “RFP 문의” 등 시나리오별 처리 단계 자동 생성
- **엔터프라이즈 연동**  
  - **Jira Cloud**: 티켓/이슈 자동 생성 (제목 형식: [담당부서] 의도), 일정(기한) 이슈 생성  
  - **Slack**: Bot Token으로 지정 채널에 알림  
  - **이메일**: SMTP로 자동 회신  
  - **시뮬레이션 모드**: API 키 없이도 SAP, ServiceNow 등 나머지 단계는 시뮬레이션으로 실행 가능
- **플러그인 구조**: 새 시스템(Jira, Slack, SAP, ServiceNow 등)을 베이스 클래스 상속·등록만으로 추가 가능하도록 설계

---

### 5. 기술 구성 (1장)

- **UI**: Streamlit
- **AI**: OpenAI API (이메일 분석)
- **연동**: Jira Cloud REST API, Slack API, SMTP 이메일
- **설계**: 플러그인 아키텍처(베이스 클래스 + 레지스트리), 설정은 환경변수/설정 파일

---

### 6. 기대 효과 요약 (1장)

- 분류 시간 단축, 담당자 배정 지연 제거, 시스템 입력 횟수 감소, 긴급 건 누락 감소 등 **정량적 기대 효과**를 표로 한 장에 정리해 주세요. (실제 수치는 참고용으로 “예시”라고 표기해도 됨.)

---

### 7. 마무리 (1장)

- 이 프로젝트는 **테스트·PoC·포트폴리오**용이며, 실제 사용을 위해서는 각 서비스의 API 키와 계정 설정이 필요함을 한 줄로 다시 강조
- “엔터프라이즈 환경에서 AI 기반 업무 자동화를 검증·시연하기 위한 데모”라는 포지션이 잘 드러나도록 마무리 문구 작성

---

**정리**:  
총 **7~9장 내외**의 발표 자료로 만들어 주시고, **반드시 “프로젝트 화면” 슬라이드 한 장은 가운데에 스크린샷을 넣을 빈 영역(플레이스홀더)**만 두어 주세요.  
문구는 발표자가 말하기 좋게 간결하고, 포트폴리오/데모 성격이 분명히 드러나도록 작성해 주세요.
```

---

## 사용 방법

1. 위 **「프롬프트 (복사용)」** 블록 전체를 복사합니다.
2. Gamma, Beautiful.ai, Canva AI, ChatGPT 등 **발표 자료를 생성해 주는 AI/도구**의 입력창에 붙여 넣습니다.
3. 생성된 슬라이드 중 **「프로젝트 화면」** 슬라이드의 빈 영역에, 직접 캡처한 **앱 실행 스크린샷**을 넣으면 됩니다.

## 스크린샷 추천

- **메인 화면**: 데모 이메일 선택 → 분석 결과(의도, 부서, 우선순위) 표시
- **워크플로우 실행**: 단계별 실행 결과(성공/시뮬레이션)가 보이는 화면
- **사이드바**: Jira / Slack / 이메일 연동 설정이 보이는 설정 메뉴

위 중 한 장을 「프로젝트 화면」 슬라이드에 넣으면 포트폴리오 설명에 충분합니다.
